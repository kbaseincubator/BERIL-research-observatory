# ADP1 Annotation Reassessment

Evaluating AI-generated gene function annotations (GPT-5.2 + InterProScan) for *Acinetobacter baylyi* ADP1 against existing RAST and Bakta annotations, using experimental phenotype data as ground truth.

## Authors

- Paramvir Dehal

## Research Question

Do the new AI-generated annotations improve our understanding of ADP1 biology compared to existing RAST and Bakta annotations? Without a gold-standard human-curated annotation set, we use five experimental datasets as ground truth — if a gene's annotation better explains its observed phenotype, that annotation is "better."

## Experimental Ground Truth

| Dataset | Genes | Source |
|---------|-------|--------|
| Deletion growth phenotypes | 2,034 genes × 8 carbon sources | `adp1_deletion_phenotypes` |
| TnSeq essentiality | 3,083 genes | `acinetobacter_adp1_explorer` |
| FBA predictions | 478 genes with concordance classes | `adp1_triple_essentiality` |
| Proteomics | 7 strains | `acinetobacter_adp1_explorer` |
| Aromatic catabolism network | 51 genes with subsystem assignments | `aromatic_catabolism_network` |
| Respiratory chain | 62 genes with subsystem labels | `respiratory_chain_wiring` |

## Notebooks

1. **01_data_integration.ipynb** — Build master annotation table joining all data sources
2. **02_hypothetical_resolution.ipynb** — How many hypothetical proteins does the agent resolve?
3. **03_phenotype_concordance.ipynb** — Do new annotations better explain experimental phenotypes?
4. **04_model_reconciliation.ipynb** — Can new annotations fill gaps in the FBA metabolic model?
5. **05_evidence_evaluation.ipynb** — How reliable are the agent's unique annotations? Evidence quality and fitness consistency
6. **06_source_resolution.ipynb** — Which annotation source should you trust for each gene? Evidence analysis and per-gene recommendation

## Data Collections

This project uses data from the following BERDL collections:
- `kescience_fitnessbrowser` — Fitness Browser (growth phenotypes, TnSeq essentiality)
- `kbase_ke_pangenome` — KBase pangenome (cluster assignments)

## Data

- `user_data/berdl-annotations.with-ipr.annotation_by_gene_detailed_dec_11_2025.tsv` — Agent annotations (3,083 sequences)
- `user_data/berdl_tables.db` — SQLite database with RAST, Bakta, growth, essentiality, FBA, proteomics data. This is a symlink to `projects/acinetobacter_adp1_explorer/user_data/berdl_tables.db`, which was generated by the `acinetobacter_adp1_explorer` project by querying `kescience_fitnessbrowser` and `kbase_ke_pangenome` collections, then enriching with Bakta annotations and FBA model data. To regenerate, run the data integration notebooks in that project.
- `data/master_annotation_table.csv` — Integrated master table (generated by NB01)
- `data/annotation_recommendations.csv` — Per-gene best-source recommendation (generated by NB06)

## Reproduction

**Prerequisites:** Python 3.11+ with pandas, numpy, matplotlib, seaborn, scipy, scikit-learn. No Spark required — all data is local.

**Execution order:** Run notebooks sequentially (each builds on prior outputs):
1. `notebooks/01_data_integration.ipynb` — produces `data/master_annotation_table.csv`
2. `notebooks/02_hypothetical_resolution.ipynb` — produces `data/agent_unique_annotations.csv`
3. `notebooks/03_phenotype_concordance.ipynb` — produces figures
4. `notebooks/04_model_reconciliation.ipynb` — produces figures
5. `notebooks/05_evidence_evaluation.ipynb` — evaluates agent evidence quality, produces figure
6. `notebooks/06_source_resolution.ipynb` — produces `data/annotation_recommendations.csv` and figure

**Estimated runtime:** < 2 minutes total for all six notebooks.

**Cross-project dependencies:** NB03 and NB04 load ground-truth data from four sibling projects. These must be present in the repository:
- `projects/adp1_deletion_phenotypes/data/condition_specific_genes.csv` — condition-specific growth phenotypes
- `projects/aromatic_catabolism_network/data/final_network_model.csv` — aromatic pathway gene assignments
- `projects/respiratory_chain_wiring/data/respiratory_chain_genes.csv` — respiratory chain subsystem labels
- `projects/adp1_triple_essentiality/data/triple_gene_table.csv` — FBA concordance classes
