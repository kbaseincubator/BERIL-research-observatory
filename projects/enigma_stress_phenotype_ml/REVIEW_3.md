## Suggestions

1. **[Important] Add `scripts/run_regression_only.py` to the README Reproduction section.** NB05 Cell 9 is empty — the regression models powering Finding 1 were generated by the standalone script, not the notebook. Update the README Reproduction section to:
   ```bash
   jupyter notebook notebooks/05_Model_Training.ipynb  # Binary classifiers (Cell 5)
   python scripts/run_regression_only.py               # Regression models (Finding 1)
   jupyter notebook notebooks/09_Isolate_Prediction.ipynb
   ```
   Without this, a user following the README instructions will reproduce binary classifier results but not the regression models.

2. **[Moderate] Add a summary print to NB06 Cell 5.** After the LOGO loop, add:
   ```python
   import pandas as pd
   logo_summary = pd.read_csv(DATA_DIR / 'adaptive_metrics.csv')
   print(logo_summary.sort_values('AUC', ascending=False)[['Stressor','AUC','Sensitivity','Specificity','F1']].to_string(index=False))
   ```
   This makes Finding 5 traceable to a notebook output without requiring the reader to load a separate CSV.

3. **[Moderate] Document which stressor-genus LOGO folds triggered single-class warnings.** NB06 Cell 2 generates 9 sklearn warning blocks about single-class folds and undefined AUC. Limitation #7 mentions that As/Pb/Ag were excluded from `adaptive_metrics.csv`, but does not characterize whether any other stressors had folds where the AUC calculation was undefined (e.g., sparse metals with very few positives per genus, where some genera had zero positives). Either add a diagnostic cell that counts single-class folds per stressor, or add a Limitation note: "For metal stressors with few positive examples per genus, some LOGO folds contained a single class (no positives); AUC in these folds was undefined and excluded from the per-stressor average."

4. **[Minor] Add runtime estimates to the README Reproduction section.** NB03 (ESM-2 embeddings for 215K proteins) requires GPU or high-memory CPU — likely multi-hour on CPU. `scripts/run_regression_only.py` (11 metals × CatBoost regression + GroupShuffleSplit) takes tens of minutes on CPU. NB06 (49 stressors × LOGO, ~26 minutes end-to-end based on log timestamps 19:52–20:18) is the longest notebook. Stating these estimates prevents surprises.
